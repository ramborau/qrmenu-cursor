# Agents Directory

This directory contains configuration files for each specialized agent in the QR Menu project.

## Available Agents

1. **[Frontend Agent](./frontend-agent.md)** - UI development, React components, and client-side functionality
2. **[Backend Agent](./backend-agent.md)** - API routes, server logic, and external service integrations
3. **[Database Agent](./database-agent.md)** - Schema design, migrations, and data modeling
4. **[Testing Agent](./testing-agent.md)** - Test suites, quality assurance, and CI/CD automation
5. **[Architect Agent](./architect-agent.md)** - System design, infrastructure, and technical strategy

## Usage

Each agent file contains:
- Role definition
- Detailed responsibilities
- Technical stack and focus areas
- **Model Context Protocol (MCP) References** - Links to comprehensive documentation
- Expected outputs/deliverables
- Code quality standards
- Collaboration guidelines

## Model Context Protocol (MCP) Integration

All agents are integrated with MCP references from `../MCP.MD`:
- **Frontend Agent:** Next.js, React, ShadCN/UI MCPs
- **Backend Agent:** Next.js, Better Auth MCPs
- **Database Agent:** Prisma MCP
- **Testing Agent:** All MCPs (for testing context)
- **Architect Agent:** All MCPs (for architectural decisions)

Refer to the specific agent file when working in that domain or when assigning tasks to AI agents. Each agent includes relevant MCP links for accessing comprehensive technology documentation.
